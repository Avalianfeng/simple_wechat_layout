## 1. 产品目标（我们要做什么）

做一个**极简、可自托管**的微信公众号排版服务，优先给家人（妈妈）方便使用。

- **排版目标**：简单可读即可，不追求复杂美学；「有总比没有好」
- **输入**：不强制用户写 Markdown；可粘贴普通文章/文本，系统再整理
- **输出**：微信公众号可识别的**内联样式 HTML**（可复制粘贴；后续可选推草稿）
- **形态**：自托管小服务 + 超简网页；**不做 iOS 原生 App**（手机用浏览器即可）

---

## 2. 明确不做什么

| 不做 | 原因 |
|------|------|
| 以 `md2wechat-skill` 为主干 | 好看排版依赖远程 API；对「简单排版 + 自托管」过重 |
| Fork 并深度改造 `doocs/md` 整仓 | 它是完整编辑器产品，改 UI/流程成本高、升级痛苦 |
| 从零发明微信 HTML 兼容层与全套主题 | 坑已被踩过，重复造轮子慢 |
| 上架苹果 App | 对当前目标性价比极低 |

---

## 3. 已选定技术路线

**参照 doocs/md，自建薄壳服务；借主题与渲染思路，不借整套编辑器。**

```
任意文本 / Markdown
  →（可选）LLM 整理成干净 Markdown
  → 本地渲染：Markdown → HTML → CSS 内联（微信兼容）
  → 套用 1～2 套简单主题（可借鉴 doocs 主题 CSS）
  → 预览 / 复制 HTML
  →（可选后期）上传图片 + 写入公众号草稿箱
```

### 3.1 从 doocs/md 借鉴什么

- Markdown → 微信兼容 HTML 的**成熟思路**
- **现成主题 CSS**（先拷贝/适配 1～2 套即可）
- 内联样式、列表/代码块/图片等**兼容经验**

### 3.2 我们自己做的部分

- 极简 Web 入口（给妈妈用）
- 自有 API（文本 in → HTML out）
- 可选：纯文本 → Markdown 的整理层
- 部署到自己的服务器
- 后期按需接微信草稿 API（可参考 md2wechat 的发布链路，但排版不依赖它）

### 3.3 与相关开源项目的关系

| 项目 | 关系 |
|------|------|
| [doocs/md](https://github.com/doocs/md) | **参照 + 借主题/渲染资产**，不是 fork 大改基座 |
| [md2wechat-skill](https://github.com/geekjourneyx/md2wechat-skill) | **不作为主干**；仅可参考草稿/上传等发布能力 |
| 小库（如 marked + juice，或 wechat-markdown-html 等） | **优先用于最快跑通渲染** |

## 4. 原则（效率优先）

1. **最快可用**：先跑通「粘贴 → 预览 → 复制到公众号」
2. **美学降级**：1 套默认主题足够；第二套可选
3. **本地闭环**：排版不依赖第三方远程排版 API
4. **输入友好**：用户不必懂 Markdown
5. **手机可用**：响应式网页即可，不追求 App

---

## 5. 建议落地顺序（开工清单草案）

1. 最小后端：`POST /convert`（markdown 或 text → html）
2. 最小前端：大文本框 + 主题选择 + 预览 + 一键复制
3. 接入 1 套主题 CSS（来自 doocs 或等价简化版）
4. （可选）LLM：纯文本 → Markdown
5. 部署到服务器，给妈妈书签打开
6. （后期）微信草稿箱 / 图片上传
